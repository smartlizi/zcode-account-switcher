# Переключатель аккаунтов ZCode (zcode-account-switcher)

**Невидимый** инструмент для переключения аккаунтов ZCode в двух вариантах: **настольное приложение (Electron GUI)** и **CLI**. Он позволяет **переключать аккаунт входа в клиент ZCode одним кликом**, без повторного прохождения OAuth и проверки по коду при каждом входе.

## Сценарии использования

- Вы часто переключаетесь между несколькими аккаунтами ZCode.
- Не хотите каждый раз повторять OAuth-вход и проверку кодом.
- Нужен настольный интерфейс для управления несколькими локальными снапшотами аккаунтов.
- Нужен CLI для автоматизации или быстрой диагностики.

## Принцип работы

Состояние входа клиента ZCode на Windows полностью хранится в этих двух файлах:

| Файл | Содержимое |
|------|------|
| `%USERPROFILE%\.zcode\v2\credentials.json` | Зашифрованные OAuth-токены в формате `enc:v1:...` (`oauth:zai:access_token` / `zcodejwttoken` / `oauth:zai:user_info`) |
| `%USERPROFILE%\.zcode\v2\config.json` | `apiKey` в каждом блоке provider (обычный JWT; payload в base64 содержит `user_id`) |

Вместе эти два файла представляют собой **полное состояние входа**. Что делает инструмент:

1. **capture**: сохраняет текущую пару файлов как «снимок аккаунта»
2. **use**: закрывает ZCode -> делает бэкап текущего состояния входа в `.last` -> **атомарно заменяет** эти два файла снимком -> перезапускает ZCode
3. Идентичность аккаунта (fingerprint) берётся из `user_id`, декодированного из JWT `apiKey` активного provider в `config.json`, и используется для дедупликации и именования

По сути переключение — это «замена файлов состояния входа + перезапуск», поэтому оно и называется **невидимым переключением**. Никаких новых QR-сканов и никаких кодов подтверждения.

## Версия CLI (командная строка · запасной вариант)

Требуется только Node.js >= 18, **без зависимостей** (не нужен `npm install`).

```cmd
cd zcode-account-switcher
node src/cli.js status
node src/cli.js quota
```

В корне репозитория также есть npm-скрипты:

```cmd
npm run status
npm run quota
npm run build
npm run desktop
```

Опционально: можно создать короткий алиас в PowerShell `$PROFILE`:
```powershell
function zcas { node "<path-to-project>\zcode-account-switcher\src\cli.js" @args }
```
После этого можно использовать `zcas status` / `zcas use 1`.

## Порядок использования

### Первый раз: один capture для каждого аккаунта

После входа в ZCode, пока нужный аккаунт уже авторизован, выполните:

```cmd
node src/cli.js capture --name "основной аккаунт"
```

Затем выйдите из ZCode -> войдите под другим аккаунтом -> снова capture:

```cmd
node src/cli.js capture --name "доп. аккаунт A"
```

Повторяйте, пока не сохраните все аккаунты.

### Ежедневное переключение

```cmd
:: посмотреть текущий аккаунт + список
node src/cli.js status

:: переключиться на аккаунт с номером 2 (ZCode закроется и перезапустится автоматически)
node src/cli.js use 2

:: или использовать id (поддерживаются префиксы, например a869)
node src/cli.js use a869
```

### Если при переключении возникла проблема

```cmd
:: откатиться к состоянию входа до переключения (используется бэкап .last)
node src/cli.js rollback
```

## Справочник команд

| Команда | Назначение |
|------|------|
| `status` | Показать текущий аккаунт и список сохранённых аккаунтов (команда по умолчанию) |
| `list` | Список всех сохранённых аккаунтов |
| `quota` | Запросить квоту текущего аккаунта (общая, использовано, остаток, разбивка) |
| `capture [--name имя] [--note заметка] [--overwrite]` | Сохранить текущее состояние входа как снимок (если снимок с таким именем уже есть, используйте `--overwrite`) |
| `use <id\|номер> [--no-restart] [--force]` | Переключить аккаунт (по умолчанию ZCode перезапускается автоматически; `--no-restart` только заменяет файлы и не запускает приложение) |
| `delete <id\|номер>` | Удалить снимок |
| `rename <id\|номер> <новое-имя>` | Переименовать |
| `rollback [--no-restart]` | Откатиться к состоянию до переключения (`.last`) |
| `kill` / `launch` | Вручную закрыть / запустить ZCode |

`<id|номер>` поддерживает номера из списка (`1`/`2`/`3`), короткий id аккаунта и префиксы id (например, `a869` соответствует `a869314f`).

## Механизмы безопасности

- **Перед переключением ZCode должен быть закрыт**: изменение файлов состояния входа во время работы клиента ненадёжно, потому что приложение может перезаписать их обратно. По умолчанию инструмент использует `--force` и завершает процесс автоматически; без этого переключение отклоняется и показывается подсказка.
- **Автобэкап**: перед каждым `use` текущие два файла копируются в `.last/`, поэтому `rollback` всегда доступен.
- **Атомарная запись**: файлы сначала пишутся в `.tmp`, а потом переименовываются, чтобы не получить «полузаписанное» состояние входа.
- **Самовосстановление при ошибке записи**: если замена не удалась, инструмент автоматически восстанавливает состояние из `.last` и не оставляет повреждённое состояние.
- **Дедупликация по fingerprint**: повторный `capture` для того же аккаунта сообщит, что он уже существует (`--overwrite` нужен для замены).

## Частые вопросы

**Q: После переключения ZCode всё ещё показывает старый аккаунт?**  
A: Убедитесь, что ZCode был закрыт и перезапущен в процессе переключения (если `status` показывает «running», `use` автоматически сделает kill + launch). Если используется `--no-restart`, нужно вручную перезапустить ZCode, чтобы изменения вступили в силу.

**Q: `status` показывает «Unable to recognize current signed-in account»?**  
A: Это значит, что в текущем `config.json` нет provider с открытым JWT `apiKey` (всё может быть зашифровано как `enc:` или аккаунт не вошёл в систему). Переключение всё равно возможно, потому что инструмент использует целиком файлы из снимка, но fingerprint откатится к более слабому хешу. Рекомендуется выполнять capture для каждого аккаунта, когда включён provider с открытым `apiKey`.

**Q: Токены в снимке аккаунта истекают?**  
A: Да. У `access_token` есть срок действия. Но после запуска клиент ZCode сам продлевает его через refresh-механизм, поэтому снимок в основном сохраняет «идентичность входа», а продление делает клиент. Если аккаунт долго не использовался и refresh тоже истёк, просто войдите снова и выполните `capture --overwrite`.

**Q: Что делать при смене компьютера?**  
A: Просто скопируйте весь каталог `accounts/` (при условии, что на новой машине тоже установлен ZCode и путь `.zcode/v2` совпадает).
